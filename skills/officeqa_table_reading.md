# OfficeQA Table Reading Guide

Treasury Bulletin tables can be dense and easy to misread. Use this guide whenever extracting values.

## 1. Read the table structure carefully
Before selecting any number, identify:
- the table title
- the table subject/metric
- row labels
- column labels
- units
- footnotes or notes
- whether the table continues from a previous page

Do not assume the closest number is the correct one.

## 2. Watch for header complexity
Treasury tables may use:
- multi-row headers
- grouped column headers
- merged categories
- date labels above subcolumns
- monthly, quarterly, yearly, or fiscal breakdowns

Always resolve the full header path before choosing a value.

Example:
A column may look like:
- Fiscal year 1952
  - Q1
  - Q2
  - Total

Make sure you choose the requested level.

## 3. Distinguish totals from components
Many tables contain:
- grand totals
- subtotals
- category rows
- subcategory rows

Do not confuse:
- a category total with one of its components
- a subtotal with the overall total
- a monthly figure with an annual total

## 4. Confirm the period type
Questions may refer to:
- calendar year
- fiscal year
- monthly value
- quarterly value
- end-of-period value
- average over a period

Do not treat these as interchangeable.

## 5. Confirm units
Units may appear in:
- the title
- a subtitle
- a column header
- a footnote

Common units:
- dollars
- thousands of dollars
- millions of dollars
- billions of dollars
- percent

A correct raw number with the wrong unit is still a wrong answer.

## 6. Check table continuations
A table may continue across:
- multiple pages
- multiple sections
- multiple document parts

Make sure the row and column you use come from the same table context.

## 7. Be careful with visual proximity
Nearby numbers are often misleading.
Do not choose a value because it is:
- on the same page
- near the relevant row
- under a similar but different column

Use exact row-column intersection logic.

## 8. Footnotes and revised values
Some values may be:
- revised
- estimated
- preliminary
- footnoted exceptions

Read nearby notes if there is ambiguity.

## 9. Final table extraction rule
Before using any number, state internally:
- document
- table
- row
- column
- unit
- value

If any one of these is unclear, keep searching.
